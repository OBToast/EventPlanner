My submission for task 3.1C SIT 305.
This event planner utilises a room database to persistently store data on device. Events are stored as an "PlannedEvent" class which details the event title, location, and date. Events are displayed in a holder with an update and delete button, connected to corresponding functions, inside a recycler view. The app utilises a single activity, with two fragments. A fragment for adding a new event, and a fragment for viewing the list of events. You can switch between the fragments utilising a bottom navbar, or by pressing the update button on an event from the view events fragment, which will automatically take you to the add event fragment, with a bundle containing the data for the vent to prepopulate the fields. The add event fragment will then pass the new details to the event with the same ID as the event you picked before returning to the view events list.
Quite a complicated project with a lot of new moving parts, but I got there.
